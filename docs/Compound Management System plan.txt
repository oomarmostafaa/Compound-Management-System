Compound Management System - Backend Project

Build a complete production-ready backend application for a Residential Compound Management System.

═══════════════════════════════════════════════════════════════
                       TECH STACK
═══════════════════════════════════════════════════════════════

• Node.js + Express.js
• PostgreSQL + Prisma ORM
• JWT Authentication (Access Token only)
• RBAC (Role-Based Access Control)
• Cloudinary + Multer (auto-detect images & PDF)
• Nodemailer (Reset Password emails only)
• Swagger Documentation
• Docker & Docker Compose
• Joi Validation
• Bcryptjs (password hashing)
• Helmet + Rate Limiting (100 req/15min)
• Environment Variables (.env)

═══════════════════════════════════════════════════════════════
                      SYSTEM OVERVIEW
═══════════════════════════════════════════════════════════════

The system manages a residential compound with 3 roles: Admin, Resident, Staff.

┌─────────────────────────────────────────────────────────────┐
│ ADMIN                                                       │
├─────────────────────────────────────────────────────────────┤
│ • Login                                                     │
│ • Create residents + staff accounts                         │
│ • Delete residents + staff                                  │
│ • Full CRUD on buildings + apartments                       │
│ • Assign resident to apartment                              │
│ • View all requests (search + filter + pagination)          │
│ • Assign staff to requests + change status                  │
│ • View all visitors + approve/reject                        │

═══════════════════════════════════════════════════════════════
                     AUTHENTICATION
═══════════════════════════════════════════════════════════════

Implemented:
  POST /api/auth/login             → Login (returns JWT token)
  POST /api/auth/forgot-password   → Request password reset link
  POST /api/auth/reset-password    → Reset password with token

NOTES:
  • JWT Access Token only (no Refresh Tokens)
  • No Register (Admin creates accounts)
  • No Logout (stateless JWT)
  • No Account Activation email
  • No Welcome email

═══════════════════════════════════════════════════════════════
                      DATABASE MODELS
═══════════════════════════════════════════════════════════════

9 Models: User, Building, Apartment, Resident, Staff,
           Document, Request, Visitor, Announcement

Relations:
  Building    ──→ Apartment          (One-to-Many)
  Apartment   ──→ Resident           (One-to-One)
  Resident    ──→ Document           (One-to-Many)
  Resident    ──→ Request            (One-to-Many)
  Resident    ──→ Visitor            (One-to-Many)
  Staff       ──→ Request (assigned) (One-to-Many)
  User        ──→ Resident           (One-to-One)
  User        ──→ Staff              (One-to-One)

Indexes on: email, nationalId, apartment number
Transactions: Create User + Resident + Assign Apartment

═══════════════════════════════════════════════════════════════
                       FEATURES
═══════════════════════════════════════════════════════════════

• Search:      Buildings, Apartments, Residents, Requests,
               Visitors, Announcements (by name/email/number)
• Filter:      Requests (by status/type), Visitors (by status),
               Apartments (by occupancy), Announcements (by date)
• Pagination:  All listing endpoints return {page,limit,total,
               totalPages,data}
• Sorting:     Announcements (by createdAt or title, asc/desc)
• File Upload: Multer → Cloudinary (auto-detect images + PDF)
• Validation:  Joi for all request bodies
• Swagger:     Complete API docs on /api-docs
• Docker:      Dockerfile + docker-compose.yml
